## 1차 — 시각 일관성 (커밋 `d6d6460`)

랜딩 / #generate(시험지·발표자료·엑셀) / #help를 360·768·1024·1440px에서 전부 스크린샷으로 확인.

**발견·수정**
1. **360px 상단바 두 줄 깨짐** — '올인원 AI' 브랜드와 '시작하기' 버튼이 단어 중간에서 두 줄로 꺾여 상단바가 지저분했음. → 브랜드 `white-space: nowrap`, 420px 이하에서 로고 아이콘 숨김 + 여백 축소로 한 줄 수납.
2. **가－/가＋ 버튼 두 줄 표시(모바일)** — 글자 크기 버튼 내용이 세로로 쌓임. → `white-space: nowrap` + 모바일 패딩 조정.
3. **버튼 속 한국어 단어 중간 줄바꿈** — '전체 보\기', '(PDF·이미\지)' 식으로 꺾임. → 버튼·칩류에 `word-break: keep-all`.
4. **스테퍼 단위(분·장·문항)가 테두리에 밀착** — 잘려 보이는 인상. → 오른쪽 여백 추가.

**결과**: 4개 페이지 × 4개 폭 전부 `scrollWidth ≤ innerWidth`(가로 스크롤 없음) 재확인.

## 2차 — 한국어 카피 (커밋 `7b6031b`)

index.html·app.js의 사용자 노출 문자열 전수 추출 + 서버에서 내려오는 오류문까지 점검.

**발견·수정**
1. **법률 페이지 플레이스홀더 방치(치명)** — 이용약관/개인정보처리방침이 "여기에 …를 채워 주세요. (플레이스홀더)" 상태. → 무료 베타 도구에 맞는 중립적 실문안 작성(회사명 없음). 개인정보처리방침은 실제 구현과 일치하게 작성: 개인정보 미수집, 입력 내용의 Claude API 전송, 기록은 브라우저(localStorage)에만 저장, 의견·설문 익명 저장, 추적 쿠키 없음. 본문용 CSS(h2·목록·시행일)도 추가.
2. **모듈 표기 불일치** — 카드·선택상자는 'PPT', 홈·메타 문구는 '발표자료'. → 사용자 노출 라벨을 **'발표자료(PPT)'**로 통일(`src/modules.ts`의 name). 조사 처리('발표자료(PPT)를') 정상 확인.
3. **어투 불일치(~합니다 혼입)** — '완성됩니다'→'완성돼요'(히어로+og/twitter 메타), '열립니다'→'열려요'(내보내기 안내 2곳), '실제 생성 결과가 아닙니다'→'아니에요', 서버 오류문 '…너무 깁니다'→'…너무 길어요'(validation + generate/doc/exam 라우트, 테스트 정규식도 함께 갱신).
4. **시험지 전용 예시가 전 모듈에 노출** — 엑셀·발표자료 화면에도 "예: 비교급 문법을 강조해 주세요"가 보였음. → 모듈 중립 예시("예: 조금 더 쉬운 말로 써 주세요")로 교체. 기본 textarea 예시도 중립화.
5. **온보딩 안내 모호** — "5개만 골라주세요" → "예·아니요로 5개만 답해 주세요".

**건너뜀(기록)**
- 시험지 화면의 '더 부탁할 점 (선택)'(모듈 옵션)과 '더 부탁할 내용 (선택)'(공통 추가란)이 유사 중복. 입력 구조(modules.ts 가이드 스키마) 변경이 필요해 카피 수정 범위를 넘음 → 백로그 제안.

## 6차 — 성능 (커밋 `8c0715b`)

- **자산 크기**: 즉시 로딩되는 자산 중 300KB 초과 없음(최대 app.js 107KB). 무거운 번들은 이미 전부 지연 로딩 — docx.umd 1.1MB·pptxgen 452KB(내보내기 클릭 시 script 주입), hwpx 108KB·fflate 64KB(dynamic import), 디자인 썸네일 8.1MB(발표자료 디자인 목록에서만, `loading="lazy"`).

- **건너뜀(기록)**: lite 모드 잔여 CLS(예시 미리보기·자료 영역의 비동기 등장)는 스켈레톤 UI가 필요해 '저비용 수정' 범위 밖 → 백로그. `body`의 `background-attachment: fixed`는 일부 모바일에서 스크롤 비용이 있으나 시각 변화 위험이 있어 보류.
